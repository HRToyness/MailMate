# MailMate

Native macOS menu bar app that drafts replies to selected Mail.app messages via LLM APIs. Two flows:

- **Draft 3 reply options** — reads the selected message, streams three labeled variants (Short / Standard / Detailed) into a floating panel, one click pastes into Mail's reply window.
- **Dictate a reply** — press record, speak in Dutch or English, Whisper transcribes, the active LLM rewrites it into a proper reply following your rules, one click to paste.

Supports **Anthropic (Claude)** and **OpenAI (ChatGPT)** as the LLM backend — selectable in Settings. Voice transcription always uses OpenAI Whisper.

## Requirements

- macOS 14 (Sonoma) or later
- Xcode 15+ **or** Command Line Tools (build script uses `swiftc` directly)
- At least one API key:
  - Anthropic key (for Claude replies), and/or
  - OpenAI key (required for voice dictation; also works as the LLM backend)

## Build

Without Xcode (Command Line Tools only — this is what `build.sh` uses):

```bash
./build.sh                 # produces universal (arm64 + x86_64) binary
open build/MailMate.app
```

With Xcode: open `MailMate.xcodeproj` (generated by `xcodegen` from `project.yml`) and run the `MailMate` scheme. If you change `project.yml`, regenerate with `xcodegen generate`.

### Package for distribution

```bash
./build-dmg.sh             # requires build.sh to have run first
```

Produces `build/MailMate-Installer.dmg` containing the app, an Applications shortcut, a `Setup.command` that strips macOS's quarantine flag on first run, and a README. Hand someone the `.dmg` and they never need to touch Terminal.

## First-run setup

1. Menu bar envelope icon appears → click → **Settings…**
2. Choose your active provider (Anthropic or OpenAI), paste the key(s), pick a model.
3. For voice dictation: an OpenAI key is also required (Whisper only runs on OpenAI).
4. Click **Edit rules file** to tune `rules.md` with your tone and instructions.
5. On first trigger, macOS will prompt for:
   - **Automation** (Apple Events to Mail) — allow.
   - **Accessibility** (for the ⌘V paste) — toggle MailMate on in System Settings → Privacy & Security → Accessibility.
   - **Microphone** (first dictation only) — allow.
6. Optional: System Settings → Keyboard → Keyboard Shortcuts → Services — bind system-wide shortcuts to **MailMate/Draft AI reply** and/or **MailMate/Dictate AI reply**.

## Usage

Select a message in Mail, then either click the menu bar icon and pick one of:

- **Draft 3 reply options…** (⌘⇧R) — generates three variants, click Use this on the one you want.
- **Dictate a reply…** (⌘⇧D) — opens the dictation panel, press record, speak, stop, pick the cleaned reply.

## Files and storage

| What | Where |
|---|---|
| User rules (markdown) | `~/Library/Application Support/MailMate/rules.md` |
| API keys | macOS Keychain, service `com.toynessit.MailMate`, accounts `anthropic-api-key` and `openai-api-key` |
| Active provider & models | `UserDefaults` keys `provider`, `model_anthropic`, `model_openai` |
| Diagnostic log | `~/Library/Logs/MailMate.log` (auto-rotates at 1 MB) |
| Build artifacts | `build/MailMate.app` |

## Architecture

```
MailMate/
├── project.yml                    # xcodegen spec
├── build.sh                       # standalone build (Command Line Tools only)
├── MailMate.xcodeproj/            # generated from project.yml
├── docs/superpowers/specs/        # design docs for the major feature additions
└── MailMate/
    ├── MailMateApp.swift          # @main SwiftUI entry
    ├── AppDelegate.swift          # registers Services + status item + notification auth
    ├── StatusController.swift     # menu bar item + menu
    ├── ReplyDrafter.swift         # orchestrator for both flows
    ├── MailBridge.swift           # AppleScript bridge: read selection + open reply + paste
    ├── ReplyProvider.swift        # ReplyProvider protocol + ProviderKind + ProviderFactory
    ├── AnthropicClient.swift      # streaming /v1/messages client + shared prompts + VariantParser
    ├── OpenAIClient.swift         # streaming /v1/chat/completions client
    ├── WhisperClient.swift        # /v1/audio/transcriptions (multipart upload)
    ├── AudioRecorder.swift        # AVFoundation recorder (AAC m4a, level meter)
    ├── DictationPanel.swift       # dictation UI (record → transcribe → generate → pick)
    ├── VariantPanel.swift         # 3-variant picker UI
    ├── SettingsView.swift         # SwiftUI Settings (provider picker + per-provider key/model)
    ├── RulesLoader.swift          # reads/writes rules.md
    ├── KeychainHelper.swift       # per-provider Keychain save/load
    ├── Log.swift                  # serial + rotating file logger
    ├── Info.plist                 # LSUIElement, NSServices, mic/automation usage strings
    └── MailMate.entitlements      # sandbox disabled
```

Mail integration is AppleScript-based. The generated reply goes through the clipboard + synthesized ⌘V rather than setting `content` on the reply message — the latter is unreliable in recent Mail versions.

## Troubleshooting

- **Nothing pastes after picking a variant** — grant **Accessibility** permission to MailMate; every rebuild of the ad-hoc-signed binary invalidates it. System Settings → Privacy & Security → Accessibility → toggle MailMate.
- **"Set your OpenAI API key in Settings (required for voice dictation)"** — voice uses Whisper (OpenAI) even when Anthropic is your LLM. Paste both keys.
- **Service doesn't appear in the Services menu** — `/System/Library/CoreServices/pbs -flush` in Terminal.
- **Transcript wrong language** — Whisper auto-detects; occasional mix-ups on short clips. Re-record a slightly longer sentence.
- **HTTP 401** — re-save the key in Settings; Keychain writes surface failures now.
